## Contributing to Docs

When adding new documentation:

1. Place it in this `docs/` folder
2. Use clear, descriptive filenames (UPPER_CASE.md)
3. Add a link to this README
4. Include date and version info in the doc
5. Keep docs up-to-date with code changes

## Documentation Standards

- **Format**: Markdown (.md)
- **Naming**: UPPERCASE_WITH_UNDERSCORES.md
- **Structure**: Clear headings, code blocks, examples
- **Links**: Use relative paths for internal links
- **Code**: Include runnable examples where possible
